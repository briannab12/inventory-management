Data Flow — Filter Change

1
User selects a filter
FilterBar.vue v-model updates useFilters.js singleton refs (selectedLocation, selectedPeriod, selectedCategory, selectedStatus)
2
Page view reacts
Vue watch() on filter refs triggers loadData() in the active view (e.g. Inventory.vue, Orders.vue, Dashboard.vue)
3
API call dispatched
getCurrentFilters() maps frontend terms to backend params. api.js fires an axios GET with query params: ?warehouse=London&category=Sensors&month=2025-03
4
Backend filters in-memory data
main.py route handler calls apply_filters() then filter_by_month() on the pre-loaded Python lists. No database query — all filtering is in-memory list comprehension.
5
Pydantic validates & serializes
FastAPI validates each item against its Pydantic model (InventoryItem, Order, etc.) and returns a JSON array response.
6
Vue reactive ref updates
Response assigned to items.value. Computed property filteredItems recalculates. Template re-renders the table with filtered data.

API Endpoints

Method Path Filters Returns
GET /api/inventory warehouse, category List of inventory items
GET /api/inventory/{id} — Single item
GET /api/orders warehouse, category, status, month List of orders
GET /api/dashboard/summary All 4 filters KPI metrics object
GET /api/demand — Demand forecasts
GET /api/backlog — Backlog items (with has_purchase_order flag)
GET /api/spending/summary — Total costs + % changes
GET /api/spending/monthly — 12-month breakdown
GET /api/spending/categories — Per-category totals
GET /api/spending/transactions — Recent transactions
GET /api/reports/quarterly — Q1–Q4 2025 metrics
GET /api/reports/monthly-trends — Month-by-month trends
